# Руководство пользователя

## 1. Назначение приложения

Приложение предназначено для оформления заказов еды в ресторане.

В системе предусмотрены три основные роли:

- клиент;
- повар;
- администратор.

## 2. Клиент

Клиент использует приложение для просмотра меню и оформления заказа.

Основные действия клиента:

1. Открыть приложение.
2. Авторизоваться или зарегистрироваться.
3. Перейти к меню.
4. Выбрать блюда.
5. Добавить блюда в заказ.
6. Подтвердить заказ.
7. Отслеживать статус заказа.

## 3. Повар

Повар использует приложение для просмотра поступивших заказов.

Основные действия повара:

1. Открыть интерфейс повара.
2. Посмотреть список новых заказов.
3. Выбрать заказ.
4. Начать приготовление.
5. Изменить статус заказа после приготовления.

## 4. Администратор

Администратор использует приложение для управления работой системы.

Основные действия администратора:

1. Открыть интерфейс администратора.
2. Просматривать список заказов.
3. Контролировать статусы заказов.
4. Просматривать информацию о блюдах и пользователях.

## 5. Запуск приложения

Для запуска приложения используется Docker.

Команда запуска:

docker compose up --build

После запуска приложение доступно по адресу:

http://localhost:3000

Серверная часть и документация API доступны по адресу:

http://127.0.0.1:8000/docs

## 6. Остановка приложения

Для остановки приложения необходимо нажать Ctrl + C в терминале, после чего выполнить команду:

docker compose down
