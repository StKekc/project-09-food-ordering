# Тестовые сценарии проекта

## 1. Проверка запуска backend

Цель: убедиться, что серверная часть приложения запускается корректно.

Шаги:

1. Запустить проект командой docker compose up --build.
2. Открыть http://127.0.0.1:8000/docs.
3. Проверить, что открылась страница Swagger.

Ожидаемый результат:

Страница Swagger открывается без ошибки.

## 2. Проверка health-check backend

Цель: убедиться, что backend отвечает на простой проверочный запрос.

Шаги:

1. Запустить проект.
2. Выполнить команду curl http://127.0.0.1:8000/health.

Ожидаемый результат:

Backend возвращает ответ {"status":"ok"}.

## 3. Проверка запуска frontend

Цель: убедиться, что пользовательский интерфейс запускается корректно.

Шаги:

1. Запустить проект командой docker compose up --build.
2. Открыть http://localhost:3000.
3. Проверить, что открылась стартовая страница приложения.

Ожидаемый результат:

Открывается страница авторизации приложения.

## 4. Проверка регистрации пользователя

Цель: убедиться, что endpoint регистрации пользователя работает корректно.

Шаги:

1. Открыть http://127.0.0.1:8000/docs.
2. Найти endpoint POST /auth/register.
3. Нажать Try it out.
4. Ввести тестовые данные пользователя.
5. Нажать Execute.

Ожидаемый результат:

Backend возвращает успешный ответ 201 Created.

## 5. Проверка доступности frontend и backend одновременно

Цель: убедиться, что обе части приложения работают одновременно через Docker.

Шаги:

1. Запустить проект командой docker compose up --build.
2. Открыть http://localhost:3000.
3. Открыть http://127.0.0.1:8000/docs.
4. Выполнить curl http://127.0.0.1:8000/health.

Ожидаемый результат:

Frontend открывается, Swagger открывается, health-check возвращает {"status":"ok"}.

## 6. Проверка остановки проекта

Цель: убедиться, что проект корректно останавливается.

Шаги:

1. В терминале с запущенным Docker нажать Ctrl + C.
2. Выполнить docker compose down.

Ожидаемый результат:

Контейнеры frontend и backend остановлены и удалены.
